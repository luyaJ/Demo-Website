软件创新中心简介

软件创新中心于2014年10月由华东交通大学软件学院筹备建立，由占东明讲师负责带领，主要内容由学生团队进行日常管理以及运营，其核心为“以软件技术为基础，提高学生综合素养”，中心团队规模在20人左右。目前团队已经建立较为全面、初步有效的制度，能够为学生团队的组织架构、日常运营等活动提供了制度保证，并摸索到了以项目为驱动而展开的学生日常内容，养成了较为良好的技术氛围和人文氛围。


历届团队概况

第一届以优培网项目发起，并建立学生团队，采用以老师为中心的方式进行团队运营，对优培网项目进行需求调研、开发，并成功上线。第二届以老师和黄松鹏等同学一起参与运营，对学生团队的运营方式展开调整，并给予学生一定的自由度，尝试自主运营。第三届以陈希杨等同学为核心团队，实现学生自主运营，对学生团队建立初步的制度，以及对学生团队日常内容做出了初步的实践。

1.联合责任制

说明：首先是为了让每一个留任成员尽自己最大程度发挥自己的能力。也是基于学生时间有限，合理充分地调用每个人。留任的成员称为责任人。同时也是避免学生会以及社团里大家互相推卸责任的情况。

  • 责任人应当明确自己在团队中属于推动者，在团队发展中有责任推动团队发展
  • 责任人应当对团队整体状况及发展进度有自己的看法和建议，并主动与他人沟通交流
  • 责任人应当做好每一份工作，积极主动的参与团队的各种建设，理应避免逃避、拒执行的情况
  • 每次开会需履行到场的义务，对于多次不到的人员需了解情况，启动退出机制

2.传帮带制度

说明:每位责任人，对于新成员有传承团队文化及制度，帮助新人学习技术，带着新人融进团队三个方面的义务。

  • 传承团队文化及制度，每位责任人为新成员多次讲解团队文化及制度，要明确指出团队创立的初衷及好处，确保每位新成员对团队文化及制度有充分的了解和体会。
  • 帮助新成员学习技术，以师傅带徒弟的形式，以平等尊重合理沟通的方式。了解、帮助、互相提升彼此的技术水平。
  • 带新成员融进团队，做好前两项的工作。并鼓励引导新成员独立思考，明确他的需求和团队状况。不盲目融进团队。

3.项目及任务分配制度

说明：项目及任务来源于老师和学生。由学生提出课题时，需提出文档，开会讨论通过即可。项目包含指标里的项目，任务包含老师分配的项目以及任务等

  • 责任人需主动与老师和他人沟通课题及任务情况及进度，并及时向老师汇报或反馈情况。
  • 责任人需主动推进课题及任务的进度，并且有文档记录

4.例会及分享会制度

说明：例会主要内容针对于团队日常运营，分享会主要内容针对于成员之间的分享

  1. 例会:全体成员参加，并定期举行（建议一个月一次）。会议内容：团队状况，交流意见。文档记录。
  2. 分享会:由2到3名分享人，分享技术经验以及其他内容（建议两个礼拜一次)。会议内容：不定。由分享人确定。根据实际情况，确定是否要文档记录。

5.考核制度

说明：一定要设立这样的监督制度，并且明确到每位监督人员，监督情况需在群里直接说明。

  1. 每位成员需要保证在正常学习周内一周两次以上的出勤。每周考勤需要一周值日表的安排。每周进行出勤记录。每晚要说明出勤情况。有特殊情况者，在群里进行说明。
  2. 每位成员每两周将交一次“学习笔记”，学习笔记形式不限，可证明其学习的内容。需要保证团队成员的产出。
  3. 对于出勤率达不到要求的同学由设立好的监督人员进行约谈，警告。警告过，仍不出勤的同学进行被动退出。
  4. 项目和任务考核，根据实际情况进行考核。应该在项目和任务设立好时，立即针对性的写好监督要求。

6.退出机制

  1. 被动退出
    • 对于学习状况不好，无任务课题分配，与团队脱离的成员，实行被动退出。
    • 对于严重违反出勤考核制度的人，考勤4次不到，进行被动退出。
    • 大三完成交接后，整理好文档后要退出。
  2. 主动退出
    • 向老师提交申请退出书。
  1. 2月末，进行招新。应准备好招新策划及面试（笔试）文档
  2. 3月-4月，进行培训。培训应该准好培训策划以及人员安排
  3. 5月培训考核，对培训成果进行考核，准备好策划以及指标。
  4. 次年1月做好与下一届留任人员的交接情况。
  5. 每学期末需做一次总结

后期团队在进行项目分配以及考核，留任人员的重点在这里，根据每年定制好的指标，做出项目以及做好任务。做好项目计划书，项目监督情况。项目产出建议以一个月或者两个礼拜为单位。

招新背景

本中心由华东交通大学软件学院建立，“让未来更美好！”是我们每一个人追求的目标。中心目前技术人员达10多名。我们是一群有极客精神的人，我们是一群先锋者，一群有冒险精神与激情的人；我们致力于让科技、时尚、与追求完美融合进我们的理念，创造更好的产品。

招新目的

本中心在拥有课题分配制度，通过研究课题帮助学生做到从理论向实践的跨越，积累做实践项目的经验。帮助更多的学生就业的综合素质，增加软件学院的竞争力。

招新方向

软件学院大一以及其他年级，有意向参与软件创新中心的学生

招生要求

  1. 对技术开发有浓厚的兴趣，有一定的技术基础者优先。
  2. 有较强的敬业精神和责任感，勇于承担工作，不计较个人得失。
  3. 有较强的组织能力和创新能力，积极配合中心开展各项活动。
  4. 具有团队意识和团结协作精神，能够积极主动向团队靠拢。
  5. 只要你具有以上条件的任意一条即可参加。

报名方式

1)网上填表报名

2)发送简历至邮箱报名

3)面试现场报名

培训课程

  1. UI美工
  2. 前端
  3. Java
  4. 数据库
  5. Jsp等
  6. 后台逻辑
  7. 安卓

说明：各培训项目由各个培训负责人进行培训内容的安排。以入门为基础，可以适当为新成员指明学习的方向。

培训时间及周期安排

第三周 周日开始 一周2次，每个周四晚上及周日下午，周四晚上2小个小时，周日下午3个小时。为期2次。按照次数依次往后推。
